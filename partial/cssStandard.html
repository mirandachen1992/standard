CSS规范

基本设置

  • 2空格缩进
  • UTF-8 编码

空白与格式

  • 大括号与选择器之间留空，冒号后面留空。

  • 在只有一条样式时允许和选择器写到同一行。

  • 一个选择器中有多个样式声明时每条写一行。

    理由：使报错可以精确到具体的规则上，便于排错。

  • 多个选择器使用逗号隔开时写在不同的行，大括号不要另起一行。

    理由：修改时不容易漏掉逗号后面的选择器。

  • 用逗号分隔的多个样式值写成多行。

    理由：便于阅读与编辑。

  • 每条样式声明后面都加上那个分号。（最后一个值也以分号结尾）

    理由：复制起来方便,防止压缩代码出错。

  • 所有最外层引号使用双引号。

    理由：与 HTML 保持一致。

  • 对选择器的注释统一写在被注释对象的上一行。

    对属性及值的注释写于分号后。

    建议使用行注释代替块注释。必要的情况下，可以使用块注释，块状注释保持统一的缩进对齐。

    注释内外前后留空。

  • 根据属性的重要性按顺序书写

    只遵循横向顺序即可，先显示定位布局类属性，后盒模型等自身属性，最后是文本类及修饰类属性。

    显示属性 自身属性 文本属性和其他修饰
    display width font
    visibility height text-align
    position margin text-decoration
    float padding vertical-align
    clear border white-space
    list-style overflow color
    top min-width background

    如果属性间存在关联性，则不要隔开写。

  • 先写带有浏览器私有标志的，后写W3C标准的。

功能限定

  • 避免使用 ID 选择器

    理由：优先级太高，不好维护，ID选择器不可重用。

  • 禁止使用 @import 引入 CSS 文件

    理由：据说挺坑的

  • 尽量不适用Hack

    很多不兼容问题可以通过改变方法和思路来解决，并非一定需要Hack，根据经验你完全可以绕过某些兼容问题。由于浏览器自身缺陷，我们无法避开的时候，可以允许使用适当的Hack。

命名

  • 0值的单位建议省略，但不强制。

    理由：大部分0值的单位是没用的，为节省不必要的字节同时也使阅读方便，我们将0px、0em、0%等值缩写为0。

  • 除非你需要透明度而使用rgba，否则都使用#f0f0f0这样的表示方法，并尽量缩写。16进制颜色值中的字母统一小写。

    理由：切换大小写很烦，当然这样统一。

  • 类名中的字母一律小写。

    理由：它不区分大小写,但是来回切换大小写麻烦

  • 类名中只使用字母、数字以及“-”，类名建议使用破折号代替驼峰法

模块化命名

类名基本命名

  • 类名使用完整英文单词或抽掉空格的英文词组。

    驼峰写法或下划线分割单词的写法都存在一个问题，我们的主观无法判断单词的分割。比如yellowgreen这个单词，如果使用分单词的写法可能被写成yellowGreen或yello_green，造成风格不统一。当然，抽掉空格也会带来另外的问题，比如a synchronous request和asynchronous request的意思是完全相反的，这时候抽掉空格就会出问题。不过这种情况是开发者们可以主动避免的，在代码review时也可以找出存在歧义的命名。另外，在能确保准确描述目标元素的情况下，尽可能地让类名更加简短

  • 仅当有层级关系时使用“-”连接，比如组件内的元素类名采用组件名“-”子类名的形式

    这时选择器在CSS中应该平行地定义，以便降低优先级，便于覆写。
